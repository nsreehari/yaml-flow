Runners (How To Run)

Runner Quick usage Use when Notes
TS runner
TypeScript example runner		 npx tsx <example-file.ts> Running most TypeScript examples under examples/ Recommended default for TS examples
Node runner
JavaScript script runner		 node <example-file.js> Running JS examples and wrappers Used by portfolio tracker + scenario runner
CLI runner
demo-simulator-cli (root)		 node demo-simulator-cli.js <scenario.yaml> Running declarative YAML simulation scenarios — primary entry point Root-level CLI alongside board-live-cards-cli.js. Implementation in demo-simulator/
Step-machine runner
Scenario via step engine		 npx tsx demo-simulator/run-portfolio-with-step-engine.ts Running portfolio scenario through yaml-flow step-machine transitions Uses portfolio-tracker.step-flow.yaml
CLI runner
board-live-cards CLI wrapper		 node board-live-cards-cli.js <command> Interacting with board runtime directly Resolves dist/src CLI automatically
CLI runner
step-machine-cli (root)		 node step-machine-cli.js <flow.yaml> [--data <json>] Running step-machine YAML flows from the command line with inline/CLI handlers Supports JSONata input-transforms/output-transforms, handler_vars, %%PLACEHOLDER%% templating, result-mode: exit-code
