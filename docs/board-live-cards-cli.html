board-live-cards CLI Reference

Canonical command reference for board runtime initialization, card lifecycle operations, source execution, and status contracts.

Invocation

board-live-cards-cli <command> [options]

Equivalent repo wrapper usage: node board-live-cards-cli.js <command>

Commands

LiveCard JSON Structure

Each card is a JSON file that declares dependencies, data sources, and computed outputs:

{
  "id": "portfolio-value",
  "meta": { "title": "Portfolio Valuation" },
  "requires": ["holdings", "prices"],
  "provides": [{ "bindTo": "total_value", "src": "computed_values.value" }],
  "state": { "initial_data": 100 },
  "sources": [
    { "cli": "node fetch-market.js", "bindTo": "market", "outputFile": "market.json" }
  ],
  "compute": [
    { "bindTo": "value", "expr": "$sum(requires.holdings.(qty * $lookup(requires.prices, symbol).price))" }
  ],
  "view": {
    "elements": [
      { "kind": "metric", "label": "Total Value", "data": { "bind": "computed_values.value" } }
    ]
  }
}

Key fields:

  • id: unique card identifier (becomes a task in the board graph).
  • requires: array of token names that must be available before this card runs.
  • provides: expressions that bind card outputs to tokens for downstream cards; uses JSONata src.
  • state: initial data object available during compute as state.*.
  • sources: external CLI commands to fetch or compute data; results available as sources.*.
  • compute: JSONata expressions that derive new values from state, requires, sources, and earlier compute results.

Understanding JSONata in LiveCards

JSONata is a lightweight query and transformation language for JSON. In card compute steps and provides[].src, you write JSONata expressions that transform data:

// Access state properties
"state.price"

// Array operations
"$sum(state.quantities)"
"$average(requires.history.values)"

// Conditional logic
"state.quantity > 100 ? state.quantity * 0.9 : state.quantity"

// Lookup and grouping
"$lookup(requires.prices, symbol).price"
"requires.transactions{category: [$sum(amount)]}"

// Multi-step chains (access previous compute results)
"computed_values.subtotal * computed_values.discount_factor"

Learn more at jsonata.org

Composition Patterns

Sequential Data Flow: Cards depend on tokens provided by upstream cards. The board scheduler ensures dependencies are satisfied.

holdings (card) —[provides: holdings]—> portfolio-value (requires: holdings)

Compute Chaining: Later compute steps in a single card can reference results from earlier steps via computed_values.*:

"compute": [
  { "bindTo": "subtotal", "expr": "$sum(state.items.price)" },
  { "bindTo": "tax", "expr": "computed_values.subtotal * 0.1" },
  { "bindTo": "total", "expr": "computed_values.subtotal + computed_values.tax" }
]

Source Output Binding: CLI sources write JSON or text to outputFile. Use provides to expose the result as a token:

"sources": [{ "cli": "node fetch-prices.js", "bindTo": "prices", "outputFile": "prices.json" }],
"provides": [{ "bindTo": "prices", "src": "sources.prices" }]

Card Composition Checklist

  • Declare requires: Name all token dependencies explicitly.
  • Use provides for outputs: Even leaf cards should declare what they provide to downstream consumers.
  • Keep compute expressions simple: Test each JSONata expression in isolation if possible.
  • Required vs enrichment sources: By default, all sources gate card completion — the card stays in-progress and downstream cards remain blocked until every source is delivered. Set optionalForCompletionGating: true on sources that are view enrichment only: they run in the background and their data enhances the card's display (e.g. an LLM sentiment indicator, a secondary metric, a sparkline), but the card completes and unblocks downstream cards without waiting for them.
  • Timeout sources: Set timeout (ms) for long-running CLIs.
  • Test with status --json: Verify card state and topology before running the full board.

Source Execution Notes

  • source.cli is parsed with quote-aware argument splitting and executed without shell wrapping.
  • Stdout is captured and written to source output artifacts (filepath: boardDir/cardId/bindTo.json or .txt).
  • Timeout defaults to 120s unless overridden by source.timeout.
  • Sources gate completion by default (including when optionalForCompletionGating is omitted or set to false). Set optionalForCompletionGating: true for view-enrichment-only sources: they run in the background and never block task-completed or downstream cards.

Example: Multi-Card Portfolio Workflow

// cards/holdings.json
{
  "id": "holdings",
  "state": { "portfolio": [{ "symbol": "AAPL", "qty": 10 }] },
  "provides": [{ "bindTo": "holdings", "src": "state.portfolio" }]
}

// cards/price-fetch.json
{
  "id": "price-fetch",
  "requires": ["holdings"],
  "sources": [{ "cli": "node fetch-prices.js", "bindTo": "prices", "outputFile": "prices.json" }],
  "provides": [{ "bindTo": "prices", "src": "sources.prices" }]
}

// cards/portfolio-value.json
{
  "id": "portfolio-value",
  "requires": ["holdings", "prices"],
  "compute": [
    { "bindTo": "total", "expr": "$sum(requires.holdings.(qty * requires.prices[symbol].price))" }
  ],
  "provides": [{ "bindTo": "portfolio_value", "src": "computed_values.total" }]
}
