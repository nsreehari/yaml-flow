board-live-cards CLI Reference

Canonical command reference for board runtime initialization, card lifecycle operations, source execution, and status contracts.

Invocation

board-live-cards-cli <command> [options]

Equivalent repo wrapper usage: node board-live-cards-cli.js <command>

Commands

CommandPurposeExample
init <dir> [--task-executor <script>]Create or reinitialize a board runtime directory.board-live-cards-cli init ./my-board
status --rg <dir> [--json]Print human status or machine-readable status object.board-live-cards-cli status --rg ./my-board --json
add-cards --rg <dir> (--card <file> | --card-glob <glob>)Add one or many card JSON files to the board.board-live-cards-cli add-cards --rg ./my-board --card-glob "cards/*.json"
update-card --rg <dir> --card-id <id> [--restart]Reload card JSON from disk and update task config.board-live-cards-cli update-card --rg ./my-board --card-id portfolio-form --restart
remove-card --rg <dir> --id <id>Remove a card/task from the board.board-live-cards-cli remove-card --rg ./my-board --id portfolio-form
retrigger --rg <dir> --task <name>Mark task as not-started and schedule reevaluation.board-live-cards-cli retrigger --rg ./my-board --task price-fetch
task-completed --token <token> [--data <json>]Task callback for successful completion.board-live-cards-cli task-completed --token TOKEN --data '{"x":1}'
task-failed --token <token> [--error <msg>]Task callback for failure.board-live-cards-cli task-failed --token TOKEN --error "timeout"
run-sources --card <file> --token <token> --rg <dir>Execute card source entries and report delivery/failure.board-live-cards-cli run-sources --card cards/price-fetch.json --token TOKEN --rg ./my-board
run-source-fetch --in <file> --out <file> [--err <file>]Execute one source definition using source.cli.board-live-cards-cli run-source-fetch --in source.json --out out.json
process-accumulated-events --rg <dir> [--inline-loop]Drain accumulated board journal events.board-live-cards-cli process-accumulated-events --rg ./my-board

Status Contract (v1)

status --json emits a stable status object:

{
  "schema_version": "v1",
  "board": { "path": "C:/.../board" },
  "summary": { "task_count": 4 },
  "schedule": {
    "eligible": 0,
    "pending": 0,
    "blocked": 0,
    "unresolved": 0
  },
  "tasks": [
    {
      "name": "price-fetch",
      "status": "completed",
      "data_keys": ["prices"],
      "error": "optional"
    }
  ]
}

Schema file: schema/board-status.schema.json

Source Execution Notes

  • source.cli is parsed with quote-aware argument splitting and executed without shell wrapping.
  • Stdout is captured and written to source output artifacts.
  • Timeout defaults to 120s unless overridden by source.timeout.
