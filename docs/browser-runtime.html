Browser Runtime Guide

The files in browser/ are a public browser-facing runtime surface that complements yaml-flow library APIs and CLIs.

What Is Shipped

  • browser/live-cards.js: browser-side card rendering/runtime helpers
  • browser/ingest-board.js: browser-side board ingestion helpers
  • browser/card-compute.js: browser-side compute support
  • browser/live-cards.schema.json: browser-usable card schema contract

How It Fits

The browser runtime is not a separate product. It complements the other public surfaces:

  • Use npm library APIs when you want programmable embedding in Node or browser build pipelines.
  • Use CLIs when you want batteries-included server-side orchestration.
  • Use browser/* when you want cards and boards rendered or interpreted in browser environments.

The reusable live-cards / boards package layer spans server-side runtime, browser-side rendering, schema contracts, and CLI orchestration together.

Schema Relationship

browser/live-cards.schema.json mirrors the main live-cards schema contract and exists so browser consumers can validate or reason about card definitions without depending on server-only file layout.

Typical Usage Pattern

1. Server-side tooling or CLIs produce/update board and card state.
2. Browser runtime loads card definitions and board data.
3. Browser helpers render or compute card outputs for UI presentation.
